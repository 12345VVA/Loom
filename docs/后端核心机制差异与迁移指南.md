# Loom 后端与 Cool-Admin 核心机制差异及当前对齐状态

## 1. 文档定位

本文用于说明 Loom 当前如何在 FastAPI / SQLModel 生态中对齐 Cool-Admin 的核心机制。它不是待办计划，而是当前实现的机制说明和差异边界。

Loom 保留 Python 后端的实现方式，但在管理端接口、权限、菜单、响应包装和 EPS 输出上保持 Cool Admin 前端可识别的协议形态。

## 2. 权限中间件

### Loom 当前实现

Loom 使用 ASGI 中间件统一拦截 `/admin`、`/app`、`/aiapi`：

- `ScopeAuthorityMiddleware` 负责 scope 识别、白名单、Token、权限检查
- `authority_service.py` 负责 JWT 校验、Redis 缓存、URL pattern 授权
- `request.state.current_user` 挂载当前用户，服务层可继续通过 `get_current_user` 获取

权限校验流程：

1. 白名单或匿名 tag 短路
2. 提取 `Authorization: Bearer <token>`
3. 校验 token 黑名单、`password_version`、`token_version`、会话缓存
4. 超管放行
5. 普通用户按 `permission -> URL pattern` 授权

### 与 Cool-Admin 的差异

Cool-Admin Midway 更依赖 IoC、装饰器反射和上下文注入。Loom 使用 FastAPI route meta 与模块注册表实现同等语义，不依赖 TypeScript 反射。

## 3. 权限体系

Loom 权限点使用：

```text
{module}:{resource}:{action}
```

示例：

- `base:sys:user:add`
- `base:sys:menu:export`
- `task:info:start`

运行时会将权限点映射为 `METHOD /admin/...` URL pattern，缓存到 Redis，供中间件快速判断。前端仍使用同一权限字符串驱动 `v-permission`。

## 4. 自动路由与 CRUD

Loom 通过模块目录和控制器元数据生成路由：

```text
backend/app/modules/<module>/controller/<scope>/<resource>.py
```

`CoolControllerMeta` 可为管理端资源自动生成：

- `/list`，GET/POST 双协议，POST 为主协议
- `/page`，GET/POST 双协议，POST 为主协议
- `/info`
- `/add`
- `/update`
- `/delete`

同时自动登记：

- 权限点
- URL 授权规则
- EPS 路由元数据

自定义动作通过 `@Get` / `@Post` 声明，若需要按钮权限，必须显式设置 `permission`。

## 5. EPS

Loom 已提供 `/admin/base/open/eps`，用于输出 Cool Admin 前端可识别的模块、控制器、方法和字段元数据。

当前约定：

- `source`: 后端模型字段名，保持 `snake_case`
- `prop` / `propertyName`: 前端公开字段名，经过 `resolve_alias`
- 查询元数据输出前端请求参数，例如 `taskType`、`orderNum`、`createTime`
- CRUD `page/list` 在 EPS 中仍以 POST 作为主协议

这不是简单“伪造 OpenAPI”，而是结合控制器注册表、模型扫描和 FastAPI OpenAPI 信息生成的框架协议层。

## 6. 响应包装

`ResponseEnvelopeMiddleware` 将 JSON 响应包装成 Cool Admin 风格：

```json
{
  "code": 1000,
  "message": "success",
  "data": {}
}
```

分页结构会规范为：

```json
{
  "list": [],
  "pagination": {
    "page": 1,
    "size": 20,
    "total": 100
  }
}
```

为避免大响应内存风险，超过 `RESPONSE_ENVELOPE_MAX_BYTES` 的 JSON 响应会跳过包装。

## 7. 模块化

每个模块通过 `config.py` 暴露 `MODULE_CONFIG`，声明：

- 模块名、标签、描述
- scope
- bootstrap
- 白名单
- 全局中间件和模块中间件
- 菜单清单
- 兼容权限配置

模块加载由 `loader.py` 自动发现。`MODULE_LOAD_STRICT=True` 时加载失败会中断启动；关闭后会记录错误并跳过失败模块。

## 8. 当前边界

- Python 没有 Midway 的 IoC/反射生态，因此 Loom 通过显式元数据对象承载控制器声明
- EPS 已覆盖当前前端需要的字段与接口元数据，但不是 TypeScript 原生反射生成
- 进程内事件总线已提供，跨进程事件仍建议接入 Redis Stream、Celery 或消息队列
- Alembic 骨架已存在，生产接入时仍需创建 baseline revision
