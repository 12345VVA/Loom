# 通知系统框架说明

Loom 通知系统第一版以站内通知为核心，覆盖系统通知、业务通知和任务通知。通知统一落库，不直接接入 WebSocket、邮件、短信或 Webhook；后续可在服务层扩展投递 provider。

## 核心模型

- `notification_message`：通知主体，包含标题、内容、类型、级别、来源模块、业务键、跳转链接、发送状态、计划发送时间和过期时间。
- `notification_recipient`：用户接收记录，包含用户、角色、部门、租户、已读、归档和删除状态。
- `notification_template`：通知模板，支持标题模板、内容模板、默认级别和默认跳转链接。
- `notification_rule`：受众规则，支持用户、角色、部门、租户、全体管理员和安全条件。

## 服务入口

`NotificationService` 提供统一发送能力：

```python
NotificationService(session).send_system(...)
NotificationService(session).send_business(...)
NotificationService(session).send_task(...)
NotificationService(session).render_template(code, context)
NotificationService(session).resolve_recipients(rule)
```

受众规则可传 `AudienceRule`、`dict` 或 JSON 字符串。系统会解析用户、角色、部门和全体管理员，并按用户去重。

## 管理端接口

- `/admin/notification/message/page|list|info|add|update|delete`
- `/admin/notification/message/mine`
- `/admin/notification/message/unreadCount`
- `/admin/notification/message/read`
- `/admin/notification/message/readAll`
- `/admin/notification/message/archive`
- `/admin/notification/template/page|list|info|add|update|delete`
- `/admin/notification/rule/page|list|info|add|update|delete`

普通用户读取自己的通知；管理员通过菜单权限管理通知、模板和规则。

## 任务通知

`TaskInfo` 增加通知配置：

- `notify_enabled`
- `notify_on_success`
- `notify_on_failure`
- `notify_on_timeout`
- `notify_recipients`
- `notify_template_code`
- `notify_timeout_ms`

任务执行完成后，系统根据任务配置判断是否发送通知。耗时任务以 `consume_time >= notify_timeout_ms` 判定超时通知，不改变任务成功/失败状态。

## 当前边界

- 第一版只做站内通知落库。
- `condition` 只支持框架白名单枚举，不执行任意表达式。
- 租户受众字段已预留；如果项目未启用完整租户机制，暂不解析租户用户。
- 顶栏未读角标可通过 `unreadCount` 轮询接入。
