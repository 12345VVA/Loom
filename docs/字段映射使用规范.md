# 字段映射使用规范

> 版本：v1.0
> 生效日期：2026-04-21
> 适用范围：所有后端接口开发

---

## 一、概述

本项目后端使用 Python **snake_case** 命名规范，前端使用 JavaScript **camelCase** 命名规范。为确保前后端数据交互的一致性，必须遵循本规范进行字段映射转换。

### 1.1 核心原则

1. **零手动映射** - 禁止在 Service 层手动进行字段名转换
2. **语义集中** - 所有语义映射统一在 `FIELD_ALIASES` 中管理
3. **类型转换归模型** - 类型转换（如 bool ↔ int）统一在 Pydantic 模型层处理
4. **双向兼容** - 使用 `populate_by_name=True` 确保前后端都能正确解析

### 1.2 转换架构

```
┌─────────────────────────────────────────────────────────────┐
│                     字段映射转换架构                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  前端 (camelCase)                                          │
│       ↓                                                     │
│  Request 模型 (snake_case + validation_alias)              │
│       ↓                                                     │
│  Service 处理 (snake_case)                                 │
│       ↓                                                     │
│  数据库 (snake_case)                                       │
│       ↓                                                     │
│  SQLModel (snake_case)                                     │
│       ↓                                                     │
│  Response 模型 (snake_case + serialization_alias)          │
│       ↓                                                     │
│  前端 (camelCase)                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

EPS 元数据沿用同一映射：`source` 表示后端 `snake_case` 源字段，`prop` / `propertyName` 表示前端公开字段名。

## 二、命名转换规则

### 2.1 纯命名转换

对于纯命名风格转换的字段，使用 `to_camel()` 函数自动处理：

| 数据库字段 (snake_case) | 前端字段 (camelCase) | 转换规则 |
|------------------------|---------------------|----------|
| `user_id` | `userId` | 首单词小写，后续单词首字母大写 |
| `nick_name` | `nickName` | 同上 |
| `head_img` | `headImg` | 同上 |
| `is_show` | `isShow` | 同上 |
| `keep_alive` | `keepAlive` | 同上 |
| `department_id` | `departmentId` | 同上 |

**实现**：无需手动配置，`resolve_alias()` 自动处理

### 2.2 语义映射字段

对于字段名完全不同或需要特殊处理的字段，必须在 `FIELD_ALIASES` 中注册：

| 数据库字段 (snake_case) | 前端字段 (camelCase) | 说明 |
|------------------------|---------------------|------|
| `sort_order` | `orderNum` | 字段名完全不同 |
| `is_active` | `status` | 字段名不同 + 类型转换 (bool → int) |
| `path` | `router` | 路由路径语义 |
| `component` | `viewPath` | 视图组件路径语义 |
| `permission` | `perms` | 权限标识缩写 |
| `full_name` | `name` | 用户姓名简称 |
| `created_at` | `createTime` | 时间字段语义 |
| `updated_at` | `updateTime` | 时间字段语义 |
| `delete_time` | `deleteTime` | 时间字段语义 |
| `password_version` | `passwordVersion` | 密码版本号 |
| `role_ids` | `roleIdList` | 列表类型语义 |
| `menu_ids` | `menuIdList` | 列表类型语义 |
| `department_ids` | `departmentIdList` | 列表类型语义 |

**实现位置**：`app/framework/api/naming.py`

```python
FIELD_ALIASES: dict[str, str] = {
    "sort_order": "orderNum",
    "is_active": "status",
    "path": "router",
    "component": "viewPath",
    "permission": "perms",
    "full_name": "name",
    "created_at": "createTime",
    "updated_at": "updateTime",
    "delete_time": "deleteTime",
    "password_version": "passwordVersion",
    "role_ids": "roleIdList",
    "menu_ids": "menuIdList",
    "department_ids": "departmentIdList",
}
```

## 三、Request 模型规范

### 3.1 基础配置

所有 Request 模型必须包含以下配置：

```python
from pydantic import BaseModel, ConfigDict

class YourCreateRequest(BaseModel):
    model_config = ConfigDict(
        populate_by_name=True,        # 允许同时接受 snake_case 和 camelCase
        alias_generator=resolve_alias, # 自动生成别名
    )
    # 字段定义...
```

### 3.2 字段命名规则

**字段名必须使用 snake_case**，与数据库字段保持一致：

```python
# ✅ 正确
class MenuCreateRequest(BaseModel):
    model_config = ConfigDict(
        populate_by_name=True,
        alias_generator=resolve_alias,
    )
    parent_id: Optional[int] = None     # snake_case
    path: Optional[str] = None          # snake_case
    sort_order: int = 0                 # snake_case
    is_active: bool = True              # snake_case

# ❌ 错误
class MenuCreateRequest(BaseModel):
    parentId: Optional[int] = None      # 不要用 camelCase
    router: Optional[str] = None        # 不要用语义别名
    orderNum: int = 0                   # 不要用语义别名
```

### 3.3 类型转换规范

类型转换统一使用 `@field_validator` 在模型层处理：

```python
from pydantic import field_validator

# bool ↔ int 转换 (is_active → status)
class YourCreateRequest(BaseModel):
    model_config = ConfigDict(
        populate_by_name=True,
        alias_generator=resolve_alias,
    )
    is_active: bool = True

    @field_validator("is_active", mode="before")
    @classmethod
    def parse_status(cls, v):
        """接受前端传来的 status (0/1) 或 is_active (true/false)"""
        if isinstance(v, int):
            return v == 1
        return bool(v)

# 字符串类型转换 (menu type)
class MenuCreateRequest(BaseModel):
    model_config = ConfigDict(
        populate_by_name=True,
        alias_generator=resolve_alias,
    )
    type: str = "button"

    @field_validator("type", mode="before")
    @classmethod
    def normalize_type(cls, v):
        """将 "group"/"menu"/"button" 规范化为内部格式"""
        if v in (0, "0", "group", "dir"):
            return "group"
        if v in (1, "1", "menu"):
            return "menu"
        return "button"
```

### 3.4 完整示例

```python
from pydantic import BaseModel, ConfigDict, field_validator
from typing import Optional
from app.framework.api.naming import resolve_alias

class UserCreateRequest(BaseModel):
    """用户创建请求"""
    model_config = ConfigDict(
        populate_by_name=True,
        alias_generator=resolve_alias,
    )

    username: str
    full_name: str
    nick_name: str = ""
    password: str
    head_img: Optional[str] = None
    email: Optional[str] = None
    phone: Optional[str] = None
    remark: Optional[str] = None
    department_id: Optional[int] = None
    role_ids: list[int] = []
    is_active: bool = True

    @field_validator("username", "password", mode="before")
    @classmethod
    def strip_whitespace(cls, v: str) -> str:
        """去除用户名和密码的前后空格"""
        if isinstance(v, str):
            return v.strip()
        return v

    @field_validator("is_active", mode="before")
    @classmethod
    def parse_status(cls, v):
        """接受 status (0/1) 或 is_active (true/false)"""
        if isinstance(v, int):
            return v == 1
        return bool(v)
```

**前端调用示例**：

```javascript
// 前端可以使用 camelCase (自动映射)
const data = {
  username: "admin",
  fullName: "系统管理员",    // → full_name
  nickName: "Admin",        // → nick_name
  headImg: "/avatar.png",   // → head_img
  departmentId: 1,          // → department_id
  roleIdList: [1, 2],       // → role_ids
  status: 1                 // → is_active (bool)
};
```

### 3.5 禁止的做法

```python
# ❌ 禁止：使用 AliasChoices 手动映射
class MenuCreateRequest(BaseModel):
    parent_id: Optional[int] = PydanticField(
        validation_alias=AliasChoices("parentId", "parent_id")
    )

# ❌ 禁止：直接使用 camelCase 字段名
class UserCreateRequest(BaseModel):
    fullName: str  # 应该用 full_name

# ❌ 禁止：在 Service 中手动 pop 字段名
def _before_add(self, data: dict) -> dict:
    data["parent_id"] = data.pop("parentId", None)  # 不要这样做
```

## 四、Response 模型规范

### 4.1 基础配置

所有 Response 模型必须包含以下配置：

```python
from pydantic import BaseModel, ConfigDict

class YourRead(BaseModel):
    model_config = ConfigDict(
        from_attributes=True,       # 支持从 ORM 模型创建
        populate_by_name=True,       # 允许内部使用 snake_case
        alias_generator=resolve_alias, # 自动生成序列化别名
    )
    # 字段定义...
```

### 4.2 字段命名规则

**字段名必须使用 snake_case**，序列化时自动转换为 camelCase：

```python
# ✅ 正确
class MenuRead(BaseModel):
    model_config = ConfigDict(
        from_attributes=True,
        populate_by_name=True,
        alias_generator=resolve_alias,
    )
    parent_id: Optional[int] = None       # 序列化为 parentId
    path: Optional[str] = None            # 序列化为 router
    sort_order: int                       # 序列化为 orderNum
    is_active: bool = True                # 序列化为 status (int)

# ❌ 错误
class MenuRead(BaseModel):
    parentId: Optional[int] = None        # 不要用 camelCase
    router: Optional[str] = None          # 不要用语义别名
```

### 4.3 类型转换规范

类型转换统一使用 `@field_serializer` 在模型层处理：

```python
from pydantic import field_serializer

# bool → int 转换 (is_active → status)
class YourRead(BaseModel):
    model_config = ConfigDict(
        from_attributes=True,
        populate_by_name=True,
        alias_generator=resolve_alias,
    )
    is_active: bool = True

    @field_serializer("is_active")
    def serialize_status(self, v: bool) -> int:
        """将 bool 转换为前端期望的 0/1"""
        return 1 if v else 0

# datetime 格式化
class YourRead(BaseModel):
    created_at: datetime

    @field_serializer("created_at")
    def serialize_created_at(self, v: datetime) -> str:
        """格式化时间为 ISO 字符串"""
        return v.isoformat() if v else None
```

## 五、Service 层规范

### 5.1 基础原则

**Service 层禁止进行字段名转换**，所有字段统一使用 snake_case。

系统兼容模型也遵循同一规则：`SysParamRead`、`SysLoginLogRead`、`SysSecurityLogRead` 等模型内部字段使用 `key_name`、`login_type`、`operator_id`，对外通过 `resolve_alias` 输出 `keyName`、`loginType`、`operatorId`。

### 5.2 _row_to_dict 规范

`_row_to_dict` 方法仅用于提取和补充业务数据，不进行字段名转换：

```python
# ✅ 正确：只补充业务数据
def _row_to_dict(self, row: Any) -> dict:
    data = super()._row_to_dict(row)
    
    # 补充业务字段
    user_id = data.get("id")
    if user_id:
        roles = get_user_roles(self.session, user_id)
        data["role_ids"] = [role.id for role in roles]  # snake_case
        data["role_name"] = ",".join(role.name for role in roles)
    
    return data

# ❌ 错误：进行字段名转换
def _row_to_dict(self, row: Any) -> dict:
    data = super()._row_to_dict(row)
    
    # 不要做字段名转换！
    data["roleIdList"] = data.pop("role_ids", [])  # ❌
    data["createTime"] = data["created_at"]         # ❌
    data["status"] = 1 if data["is_active"] else 0  # ❌
    
    return data
```

### 5.3 _before_add/_before_update 规范

钩子方法中禁止进行字段名转换：

```python
# ✅ 正确：纯业务逻辑
def _before_add(self, data: dict) -> dict:
    # 业务验证
    existing = self.session.exec(
        select(User).where(User.username == data.get("username"))
    ).first()
    if existing:
        raise HTTPException(status_code=409, detail="用户名已存在")
    
    # 类型规范化
    data["type"] = self._normalize_menu_type(data.get("type"))
    
    return data

# ❌ 错误：手动转换字段名
def _before_add(self, data: dict) -> dict:
    # 不要手动 pop 字段名！
    data["parent_id"] = data.pop("parentId", None)    # ❌
    data["path"] = data.pop("router", None)           # ❌
    data["sort_order"] = data.pop("orderNum", 0)      # ❌
    data["is_active"] = int(data.pop("status", 1)) == 1  # ❌
    
    return data
```

### 5.4 返回数据规范

Service 方法返回的数据格式：

| 方法 | 返回类型 | 字段格式 | 说明 |
|------|----------|----------|------|
| `list()` | `list[dict]` 或 `list[Model]` | camelCase | 通过 `_finalize_data()` 或模型自动转换 |
| `page()` | `PageResult[dict]` | camelCase | 通过 `_finalize_data()` 转换 |
| `info()` | `dict` 或 `Model` | camelCase | 通过 `_finalize_data()` 或模型自动转换 |
| `add()` | `Model` | camelCase | 返回模型实例，框架自动序列化 |
| `update()` | `Model` | camelCase | 返回模型实例，框架自动序列化 |

## 六、特殊场景处理

### 6.1 联合主键或复合字段

对于包含多个值的字段，使用列表类型：

```python
class RoleAssignRequest(BaseModel):
    model_config = ConfigDict(
        populate_by_name=True,
        alias_generator=resolve_alias,

    menu_ids: list[int] = []    # 前端: menuIdList
    department_ids: list[int] = []  # 前端: departmentIdList
```

### 6.2 嵌套对象

对于嵌套对象，使用子模型：

```python
class DepartmentInfo(BaseModel):
    model_config = ConfigDict(
        populate_by_name=True,
        alias_generator=resolve_alias,

    id: int
    name: str
    parent_id: Optional[int] = None

    department: Optional[DepartmentInfo] = None  # 嵌套对象
```

### 6.3 动态字段

对于需要动态计算的字段，使用 `@computed_field`：

```python
from pydantic import computed_field

class UserRead(BaseModel):
    model_config = ConfigDict(
        from_attributes=True,
        populate_by_name=True,
        alias_generator=resolve_alias,
    )
    first_name: str
    last_name: str

    @computed_field
    @property
    def full_name(self) -> str:
        """动态计算的全名字段"""
        return f"{self.first_name} {self.last_name}"
```

```python
from enum import Enum
```

## 八、常见问题

### 8.1 前端收到的字段仍是 snake_case？

**可能原因**：
1. Response 模型未配置 `alias_generator=resolve_alias`
2. 使用了 `model_dump()` 而不是 `model_dump(by_alias=True)`
3. 直接返回 dict 而不是 Pydantic 模型

**解决方法**：
```python
# ✅ 正确
return YourRead.model_validate(entity)  # FastAPI 自动序列化

# ✅ 正确（需要 dict）
return entity.model_dump(by_alias=True)

# ❌ 错误
return entity.model_dump()  # 没有 by_alias=True
```

### 8.2 前端发送的数据后端无法识别？

**可能原因**：
1. Request 模型未配置 `populate_by_name=True`
2. Request 模型未配置 `alias_generator=resolve_alias`

**解决方法**：
```python
# ✅ 正确配置
class YourCreateRequest(BaseModel):
    model_config = ConfigDict(
        populate_by_name=True,        # 必须配置
        alias_generator=resolve_alias, # 必须配置
    )
```

### 8.3 类型转换不生效？

**可能原因**：
1. `@field_validator` 或 `@field_serializer` 未正确装饰
2. mode 参数设置不正确

**解决方法**：
```python
# ✅ 正确
@field_validator("is_active", mode="before")  # 输入转换
@classmethod
def parse_status(cls, v):
    if isinstance(v, int):
        return v == 1
    return bool(v)

# ✅ 正确
@field_serializer("is_active")  # 输出转换
def serialize_status(self, v: bool) -> int:
    return 1 if v else 0
```

### 8.4 子类字段映射不生效？

**可能原因**：子类覆盖了 `model_config` 但未继承父类配置

**解决方法**：
```python
# ✅ 正确：子类自动继承
class ChildModel(ParentModel):
    new_field: str = "value"
    # 不需要重新定义 model_config

# ❌ 错误：覆盖了父类配置
class ChildModel(ParentModel):
    model_config = ConfigDict()  # 这会覆盖父类配置！
```

## 九、代码审查检查点

### 9.1 PR 提交前检查

在提交代码前，请确认：

1. **无手动字段转换**：代码中不存在 `data["camelCase"] = data.pop("snake_case")`
2. **模型配置完整**：所有 Request/Response 模型已配置 `alias_generator`
3. **语义映射注册**：新增的语义字段已在 `FIELD_ALIASES` 中注册
4. **类型转换在模型层**：所有类型转换使用 `@field_validator` 或 `@field_serializer`
5. **测试覆盖**：已编写测试验证字段映射正确性

### 9.2 Code Review 检查点

审查代码时，重点检查：

- [ ] Request/Response 模型是否配置了 `alias_generator=resolve_alias`
- [ ] Request/Response 模型是否配置了 `populate_by_name=True`
- [ ] 字段名是否使用 snake_case
- [ ] Service 层是否存在手动字段名转换
- [ ] 类型转换是否在模型层处理
- [ ] 新增语义字段是否在 `FIELD_ALIASES` 中注册

```python
from pydantic import BaseModel, ConfigDict, field_validator
from typing import Optional
from app.framework.api.naming import resolve_alias

class YourCreateRequest(BaseModel):
    model_config = ConfigDict(
        populate_by_name=True,
        alias_generator=resolve_alias,
    )
    
    # 字段定义
    name: str
    description: Optional[str] = None
    
    # 字段验证
    @field_validator("name", mode="before")
    @classmethod
    def strip_name(cls, v: str) -> str:
```

```python
from pydantic import BaseModel, ConfigDict, field_serializer
```
