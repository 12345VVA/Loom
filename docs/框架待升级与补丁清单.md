# 框架待升级与补丁清单

> 状态口径：`[已修复]` 表示已有代码或测试覆盖；`[部分修复]` 表示核心风险已收敛但仍有增强项；`[待修复]` 表示尚未落地；`[暂缓]` 表示需要产品/部署侧决策或大版本投入。

## 一、安全类

### 1. 文件上传无校验 [已修复]
- 代码证据：`backend/app/framework/storage.py` 的 `validate_upload()` 校验扩展名和大小；`LocalStorageProvider._resolve_safe_path()` 统一防路径遍历。
- 验证方式：`backend/tests/test_framework_patch_list.py::test_local_storage_rejects_path_traversal_delete`。

### 2. 无 API 限流 [已修复]
- 代码证据：`RateLimitMiddleware` 基于缓存计数器限制 `/admin`、开放接口和通用接口。
- 剩余动作：后续可按用户 ID、租户、接口权重细化。

### 3. CORS 过于宽松 [已修复]
- 代码证据：`CORS_ALLOW_METHODS`、`CORS_ALLOW_HEADERS`、`CORS_ORIGINS` 统一进入 `CORSMiddleware`。
- 验证方式：检查 `backend/main.py` 不再使用 `allow_methods=["*"]`、`allow_headers=["*"]`。

### 4. 密码存储算法可升级 [已修复]
- 代码证据：`hash_password()` 使用可配置 `PASSWORD_PBKDF2_ITERATIONS`；`password_needs_rehash()` 支持登录后静默升级旧哈希。
- 验证方式：`test_password_hash_rehash_policy`。

### 5. 默认管理员未强制改密 [已修复]
- 合并说明：原 #5 与 #42 为同一问题。
- 代码证据：`User.password_changed_at`、登录响应 `forcePasswordChange`、默认管理员首次登录强制改密。

### 6. CSRF 防护 [部分修复]
- 代码证据：`AdminCsrfOriginMiddleware` 支持 `ADMIN_CSRF_ORIGIN_CHECK_ENABLED`，对管理端变更请求校验 `Origin/Referer`。
- 剩余动作：若后续改为 Cookie 鉴权，再增加 CSRF Token 双提交机制。
- 验证方式：`test_admin_csrf_origin_check`。

## 二、架构 / 健壮性

### 7. 全局状态线程安全 [已修复]
- 代码证据：`controller_meta.py` 的控制器权限/路由注册表使用 `RLock`，并按 key 去重。

### 8. 操作日志中间件同步写库 [已修复]
- 代码证据：`OperationLogMiddleware` 响应生成后通过 `asyncio.to_thread()` 使用独立 Session 写入日志，失败只记录错误日志，不影响主请求。

### 9. 响应包装中间件读全量 Body [部分修复]
- 代码证据：`RESPONSE_ENVELOPE_MAX_BYTES` 超阈值跳过包装，非 JSON/文档/metrics 继续跳过。
- 剩余动作：如存在真正的长流式 JSON，需要迁移到 APIRoute 层包装。
- 验证方式：`test_response_envelope_skips_large_json`。

### 10. 模块加载缺少错误隔离 [已修复]
- 代码证据：`MODULE_LOAD_STRICT` 控制 fail-fast 或隔离记录错误。
- 验证方式：`test_module_loader_strict_mode_switch`。

### 11. BaseAdminCrudService 事务边界不清晰 [部分修复]
- 代码证据：CRUD 服务仍以单方法内提交为主，生命周期 hook 已对齐参数签名。
- 剩余动作：复杂批量操作建议补 `transaction()` 上下文 helper，避免业务服务手写多次 commit。

### 12. Session 泄漏风险 [已修复]
- 代码证据：鉴权、任务调用、操作日志均使用 `with Session(engine)`；操作日志异步写入使用独立 Session。

## 三、功能缺失

### 13. 无数据库迁移框架 [已修复]
- 代码证据：新增 `backend/alembic.ini`、`backend/alembic/env.py`、`backend/alembic/script.py.mako`。
- 剩余动作：生产首次接入时执行 baseline revision，之后逐步移除 SQLite 兼容补列逻辑。

### 14. EPS 元数据不完整 [已修复]
- 代码证据：EPS 字段 `source` 保留后端字段，`prop/propertyName` 输出前端别名；查询元数据输出前端请求参数。
- 验证方式：`test_eps_uses_public_prop_and_source_field`。

### 15. 无缓存失效策略 [部分修复]
- 代码证据：权限、角色、菜单、用户变更会清理登录态缓存；新增 `cache_delete_pattern()` 与 `clear_cool_cache()`。
- 剩余动作：业务缓存需要按模块定义命名空间和失效点。

### 16. 文件存储仅本地 [已修复]
- 代码证据：`STORAGE_PROVIDER=s3/oss` 可启用 `S3StorageProvider`，支持 S3-compatible 对象存储。
- 剩余动作：如使用阿里云 OSS 原生 SDK，再单独增加厂商 Provider。

### 17. Redis 客户端过于简陋 / 生产阻塞风险 [已修复]
- 合并说明：原 #17 与 #32 为同一风险。
- 代码证据：`clear_cache_pattern()` 和 `cache_delete_pattern()` 均使用 `scan_iter()`，不再使用阻塞式 `keys()`。
- 验证方式：`test_redis_pattern_clear_uses_scan_iter`。

### 18. 查询构造器 N+1 风险 [部分修复]
- 代码证据：`QueryBuilder.apply_select()` 明确不使用 `load_only` 触发延迟加载；Relation 仍通过 join/add_columns 补充字段。
- 剩余动作：复杂对象图查询再按实体增加 `selectinload` 配置。

### 19. CoolCache 装饰器功能单一 [已修复]
- 代码证据：`CoolCache(ttl, key_prefix, namespace)` 支持命名空间，`clear_cool_cache()` 支持命名空间失效。

## 四、生产就绪

### 20. 无健康检查体系 [已修复]
- 代码证据：`/health` 返回 DB、Redis、Celery 配置检查；`/admin/base/health/ping` 返回模块级 checks。

### 21. 无监控埋点 [已修复]
- 代码证据：`MetricsMiddleware` 和 `/metrics` 提供最小 Prometheus 文本指标，受 `METRICS_ENABLED` 控制。

### 22. 日志体系不完善 [部分修复]
- 代码证据：新增 JSON 日志格式、`RequestIdMiddleware` 和 `X-Request-Id`。
- 剩余动作：生产建议接入集中日志系统并把 request_id 写入所有业务审计日志。

### 23. 无数据库连接池调优 [已修复]
- 代码证据：非 SQLite 数据库支持 `DB_POOL_SIZE`、`DB_MAX_OVERFLOW`、`DB_POOL_RECYCLE`、`DB_POOL_PRE_PING`。

## 五、扩展能力

### 24. 无国际化 [已修复]
- 代码证据：新增 `framework/i18n.py` 和 `framework/locales/zh-CN.json`。
- 剩余动作：错误响应全面接入 `t()` 仍可逐步推进。

### 25. 无数据导入导出 [部分修复]
- 代码证据：菜单已有导入导出；新增 `framework/io.py` 提供 CSV 字典导入导出基础函数。
- 剩余动作：各业务模块需决定字段白名单、权限和文件格式。

### 26. 无消息事件机制 [已修复]
- 代码证据：新增 `framework/events.py`，支持进程内 subscribe/publish。
- 剩余动作：跨进程事件需要接入 Redis Stream、Celery 或消息队列。

### 27. 无 API 版本管理 [已修复]
- 代码证据：`API_VERSION_PREFIX_ENABLED` 可额外挂载 `/api/v1`，默认关闭，不影响现有 `/admin/**`。

### 28. 审计日志不完整 [部分修复]
- 合并说明：原 #28 覆盖面较大，#38/#43 已修复敏感操作 diff 和密码重置审计。
- 代码证据：`SysSecurityLog` 支持 old/new/diff/request_id；用户、角色、菜单等敏感操作已有服务层审计入口。
- 剩余动作：继续按业务模块补齐非权限域的敏感操作。

## 六、新增漏洞与关注点

### 29. TaskInvoker 异步环境崩溃风险 [已修复]
- 代码证据：`_run_coroutine_sync()` 在已有事件循环中使用独立线程运行协程，避免 `asyncio.run()` 崩溃。
- 验证方式：`test_task_invoker_coroutine_runs_inside_existing_loop`。

### 30. LocalStorageProvider 路径遍历漏洞 [已修复]
- 合并说明：与 #1 的路径安全部分合并。
- 代码证据：`os.path.commonpath()` 校验上传根目录。

### 31. JWT 密码版本字段大小写混用 [已修复]
- 代码证据：Token payload 统一使用 `password_version`、`token_version`；字段映射只用于 API JSON。

### 32. Redis 生产阻塞风险 [已修复]
- 合并至 #17。

## 七、登录与认证安全

### 33. 缺少密码强度校验 [已修复]
- 代码证据：`validate_password_strength()` 用于用户创建、修改、重置密码。

### 34. 验证码可被重放攻击 [已修复]
- 代码证据：`captcha_check()` 验证时立即删除验证码缓存。

### 35. 登录失败日志未记录设备指纹 [已修复]
- 代码证据：登录日志记录 `device_id`，未显式传入时基于 IP、UA、语言生成稳定指纹。

### 36. 无会话并发控制 [已修复]
- 代码证据：`ADMIN_SESSION_MAX_CONCURRENT` 启用后按 token fingerprint 保留最近 N 个有效会话。
- 验证方式：`test_session_concurrency_policy`。

### 37. Token 无吊销黑名单机制 [已修复]
- 代码证据：logout 加入 JTI 黑名单；用户级吊销通过 `token_version` 实现。

### 45. 用户名与密码未去除前后空格 [已修复]
- 代码证据：登录、创建用户、更新用户、个人修改密码请求均有字段 validator。

## 八、日志管理安全

### 38. 敏感操作未记录详细变更 [已修复]
- 代码证据：`SysSecurityLogService.create_entry()` 支持 old/new/diff，并对敏感字段脱敏。

### 39. 日志写入失败被静默吞掉 [已修复]
- 代码证据：登录日志、操作日志、安全日志写入失败均记录 error 日志。

### 40. 日志表无自动清理机制 [已修复]
- 代码证据：`task.clean_expired_logs` 由 Celery Beat 每日执行。

### 41. 日志脱敏不完整 [已修复]
- 代码证据：`mask_sensitive_data()` 与安全日志脱敏覆盖密码、token、手机号、证件号、银行卡等字段。

## 九、密码管理增强

### 42. 默认管理员密码未强制首登修改 [已修复]
- 合并至 #5。

### 43. 密码重置无旧密码验证绕过风险 [已修复]
- 代码证据：个人修改密码必须验证旧密码；管理员重置密码会增加密码版本并写安全审计。

### 44. 密码哈希算法可升级 [已修复]
- 合并至 #4。
