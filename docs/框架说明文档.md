# Loom 框架说明文档

> Loom 是一个基于 **FastAPI + SQLModel + Vue 3 (cool-admin-vue)** 的全栈 AI 内容生成平台。
> 后端参考 `cool-admin-midway` 的设计理念，在 Python 生态中实现了模块化自动路由、装饰器式 CRUD 控制器、多 Scope 鉴权中间件、EPS 元数据协议等核心机制。

## 2. 模块加载机制

> 源码位置：`app/modules/loader.py`、`app/modules/module_config.py`

### 2.1 模块发现

应用启动时，`loader.py` 自动扫描 `app/modules/` 目录下所有包含 `config.py` 的子目录，将其识别为业务模块：

```
app/modules/
├── base/config.py       → 发现为基础权限模块
├── task/config.py       → 发现为系统定时任务模块
├── dict/config.py       → 发现为字典管理模块
└── demo/                → 无 config.py，不会被加载
```

每个模块必须定义 `MODULE_CONFIG`：

```python
# app/modules/base/config.py
MODULE_CONFIG = ModuleConfig(
    name="base",
    label="基础权限模块",
    description="基础的权限管理功能，包括登录、权限校验和 EPS 导出",
    order=10,
    scopes=("admin",),
    bootstrap="app.modules.base.config.bootstrap",
    middlewares=(ModuleAccessMiddleware,),       # 模块级中间件
    global_middlewares=(
        ResponseEnvelopeMiddleware,
        OperationLogMiddleware,
        AiApiAuthorityMiddleware,
        AppAuthorityMiddleware,
        AdminAuthorityMiddleware,
    ),
    config_namespace="BASE",
    init_db_file="init_db.py",
    init_menu_file="menu.json",
    admin_whitelist=(
        "/admin/base/open/login",
        "/admin/base/open/refresh",
        "/admin/base/open/refreshToken",
        "/admin/base/open/captcha",
        "/admin/base/open/eps",
        "/admin/base/health/ping",
    ),
)
```

### 2.2 ModuleConfig 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | `str` | 模块标识，用于路由前缀生成 |
| `label` | `str` | 模块显示名称 |
| `description` | `str` | 模块描述 |
| `order` | `int` | 加载优先级（越大越先加载） |
| `scopes` | `tuple[str]` | 模块支持的作用域 |
| `bootstrap` | `str \| None` | 启动初始化函数路径 |
| `middlewares` | `tuple[type]` | 模块级中间件（按前缀匹配） |
| `global_middlewares` | `tuple[type]` | 全局中间件 |
| `config_namespace` | `str \| None` | 环境变量命名空间前缀 |
| `init_db_file` | `str \| None` | 数据库初始化脚本路径 |
| `init_menu_file` | `str \| None` | 菜单清单文件路径（.json 或 .py） |
| `admin_whitelist` | `tuple[str]` | 管理端匿名白名单路径 |
| `app_whitelist` | `tuple[str]` | 应用端匿名白名单路径 |
| `aiapi_whitelist` | `tuple[str]` | AI API 端匿名白名单路径 |
| `permissions` | `tuple[PermissionConfig]` | 模块声明的权限点 |
| `admin_resources` | `tuple[ResourceConfig]` | 管理端资源定义 |

### 2.3 启动流程

`main.py` 的 `lifespan` 管理应用生命周期：

```
1. init_db()                → 创建数据库表 + SQLite 字段补齐
2. bootstrap_modules()      → 遍历模块执行初始化
   ├── 执行 init_db.py      → 初始化模块默认数据
   └── 执行 bootstrap()     → 模块自定义启动逻辑
3. load_scope_whitelists()  → 聚合所有 Scope 白名单到 app.state
4. load_global_middlewares() → 聚合并注册全局中间件（去重）
5. load_module_middlewares() → 注册模块级中间件（按前缀匹配）
6. load_module_runtime_infos() → 聚合模块运行时信息到 app.state
7. create_api_router()      → 扫描模块控制器，注册路由
8. register_compat_aliases() → 注册前端兼容路由别名
```

### 2.5 数据库初始化脚本

模块的 `init_db.py` 必须导出 `run(session)` 函数：

```python
# base 模块 — 初始化系统参数
def run(session: Session) -> None:
    if session.exec(select(SysParam).where(SysParam.key_name == "logKeep")).first() is None:
        session.add(SysParam(name="日志保存天数", key_name="logKeep", data="7", data_type=0))
    session.commit()

# dict 模块 — 初始化默认字典
def run(session: Session) -> None:
    status_type = session.exec(select(DictType).where(DictType.key == "status")).first()
    if not status_type:
        status_type = DictType(name="状态", key="status")
        session.add(status_type)
        session.commit()
        session.add(DictInfo(type_id=status_type.id, name="禁用", value="0", order_num=0))
        session.add(DictInfo(type_id=status_type.id, name="启用", value="1", order_num=1))
        session.commit()
```

## 3. 自动路由机制

> 源码位置：`app/framework/router/auto_router.py`、`app/framework/router/compat_aliases.py`

### 3.1 约定式目录映射

Loom 采用**文件路径即路由前缀**的约定，自动将控制器目录结构映射为 API 路径：

```
app/modules/<module>/controller/<scope>/<resource>.py
          ↓                ↓           ↓
    /{scope}/{module}/{resource}/*
```

支持嵌套目录（如 `admin/sys/log.py` 会映射为 `/admin/base/sys/log/*`）。

**实际映射示例：**

| 文件路径 | 生成的路由前缀 |
|---------|-------------|
| `modules/base/controller/admin/open.py` | `/admin/base/open/*` |
| `modules/base/controller/admin/comm.py` | `/admin/base/comm/*` |
| `modules/base/controller/admin/user.py` | `/admin/base/sys/user/*` |
| `modules/base/controller/admin/sys/log.py` | `/admin/base/sys/log/*` |
| `modules/task/controller/admin/info.py` | `/admin/task/info/*` |
| `modules/dict/controller/admin/type.py` | `/admin/dict/type/*` |
| `modules/base/controller/app/health.py` | `/app/base/health/*` |
| `modules/base/controller/aiapi/health.py` | `/aiapi/base/health/*` |

### 3.2 Scope 作用域

| Scope 目录名 | URL 前缀 | 说明 |
|-------------|---------|------|
| `admin` | `/admin` | 管理后台接口，需完整鉴权 |
| `app` | `/app` | 应用端接口，需 Token |
| `aiapi` | `/aiapi` | AI API 接口，需 Token |
| `open` / `public` | 无前缀 | 公开接口，无需鉴权 |

### 3.3 路由注册流程

```
1. 遍历 app/modules/*/controller/ 目录
2. 对每个 scope 子目录 (admin/app/aiapi/open/public)
3. 递归查找所有 .py 文件（支持子目录嵌套）
4. 动态 import 模块，提取 router 对象
5. 拼接前缀，注册到全局 APIRouter
6. 调用 register_compat_aliases() 注册兼容路由
```

### 3.4 兼容路由别名

`compat_aliases.py` 为 cool-admin-vue 前端提供路由兼容映射，例如：
- 提供字典数据 API 的兼容接口

## 4. 控制器装饰器与 CRUD 自动生成

> 源码位置：`app/framework/controller_meta.py`

### 4.1 @CoolController 装饰器

`@CoolController` 是 Loom 的核心装饰器，通过声明式元数据自动生成完整的 CRUD 接口：

```python
@CoolController(CoolControllerMeta(
    module="base",
    resource="sys/user",
    service=UserAdminService,
    description="用户管理",
    scope="admin",
    actions=DEFAULT_CRUD_ACTIONS,
    page_query=QueryConfig(
        keyword_like_fields=("username", "name"),
        field_eq=("status",),
        order_fields=("created_at", "updated_at", "name"),
        default_order="created_at",
    ),
    page_item_model=UserListItem,
    add_request_model=UserCreateRequest,
    update_request_model=UserUpdateRequest,
    relations=(
        RelationConfig(model=Department, column="department_id", target_column="name", alias="departmentName"),
    ),
))
class UserController(BaseController):
    pass

router = UserController.router
```

这段声明会自动生成以下 6 个接口：

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/admin/base/sys/user/list` | 获取列表 |
| POST | `/admin/base/sys/user/page` | 分页查询 |
| GET | `/admin/base/sys/user/info` | 获取详情 |
| POST | `/admin/base/sys/user/add` | 新增 |
| POST | `/admin/base/sys/user/update` | 更新 |
| POST | `/admin/base/sys/user/delete` | 删除 |

### 4.2 CoolControllerMeta 核心配置

| 字段 | 类型 | 说明 |
|------|------|------|
| `module` | `str` | 模块名，用于权限点前缀 |
| `resource` | `str` | 资源路径，如 `sys/user` |
| `service` | `type` | 服务类，应继承 `BaseCrudService` |
| `scope` | `str` | 作用域，默认 `admin` |
| `actions` | `tuple` | 要生成的 CRUD 动作，默认 `DEFAULT_CRUD_ACTIONS` (6个) |
| `page_query` | `QueryConfig` | 分页查询配置 |
| `list_query` | `QueryConfig` | 列表查询配置 |
| `soft_delete` | `bool` | 是否软删除，默认 `False` |
| `relations` | `tuple[RelationConfig]` | 关联查询配置 |
| `is_tree` | `bool` | 是否树形数据，默认 `False` |
| `parent_field` | `str` | 树形数据的父节点字段，默认 `parentId` |
| `before_hooks` | `tuple[BeforeHookConfig]` | 前置钩子 |
| `insert_params` | `tuple[InsertParamConfig]` | 参数自动注入配置 |
| `service_apis` | `tuple[ServiceApiConfig]` | 自定义服务方法路由 |
| `role_codes` | `tuple[str]` | 默认角色代码，默认 `("admin",)` |
| `name_prefix` | `str` | 权限名称前缀 |
| `code_prefix` | `str` | 权限代码前缀 |
| `info_ignore_property` | `tuple[str]` | info 响应中要排除的字段 |

### 4.3 QueryConfig 查询配置

```python
QueryConfig(
    keyword_like_fields=("username", "name"),  # 关键词模糊搜索字段
    field_eq=("status", "type"),               # 精确匹配字段
    field_like=("remark",),                    # 模糊匹配字段
    order_fields=("created_at", "name"),       # 允许排序的字段
    default_order="created_at",                # 默认排序字段
    default_sort="desc",                       # 默认排序方向
    add_order_by=(OrderByConfig(column="sort_order", direction="asc"),),  # 附加排序
    select=("id", "name", "status"),           # 选择字段（仅 EPS 导出）
    where=my_custom_where_handler,             # 自定义过滤函数
)
```

在 `BaseController` 子类中使用 `@Get`、`@Post` 装饰器声明自定义路由：

**before_hooks** — 在 CRUD 操作前执行服务方法：


## 5. 中间件与鉴权体系

> 源码位置：`app/framework/middleware/`

### 5.1 中间件执行链

请求经过的中间件链（从外到内）：

```
CORS → ModuleAccess → ResponseEnvelope → OperationLog → ScopeAuthority → 业务处理
```

各模块通过 `global_middlewares` 声明需要注册的中间件，`loader.py` 会自动聚合去重。
模块级中间件通过 `middlewares` 声明，由 `PrefixScopedMiddleware` 按前缀路由。

### 5.2 鉴权中间件

#### 统一多 Scope 鉴权 (AdminAuthorityMiddleware)

拦截 `/admin`、`/app`、`/aiapi` 三个前缀的请求，执行完整鉴权流程：

```
1. 解析请求所属 Scope（通过 URL 前缀判断）
2. 路由匹配 → resolve_request_route() 获取 endpoint
3. 读取路由元数据标签 (route_tags)
4. 调用 authorize_request() 执行鉴权
   ├── 白名单短路检查
   ├── Token 提取与 JWT 解码
   ├── passwordVersion 缓存验证
   ├── 单点登录 token 缓存验证
   └── URL 权限匹配（perms → URL 模式）
5. 鉴权通过 → 执行业务
   鉴权失败 → 返回 401/403
```

#### Scope 专用鉴权 (ScopeAuthorityMiddleware)

提供按 Scope 拆分的鉴权中间件基类，已实现三个子类：

- `AdminAuthorityMiddleware` → `/admin`
- `AppAuthorityMiddleware` → `/app`
- `AiApiAuthorityMiddleware` → `/aiapi`

区别于统一的 `AdminAuthorityMiddleware`（内部自行判断 scope），这三个子类各自只处理自己前缀的请求。

### 5.3 模块级中间件

模块通过 `middlewares` 字段声明模块级中间件，由 `PrefixScopedMiddleware` 桥接执行：

```python
# loader.py 加载时为模块中间件绑定前缀
# 例如 base 模块的 ModuleAccessMiddleware 只会处理 /admin/base/* 请求
for binding in load_module_middlewares():
    app.add_middleware(PrefixScopedMiddleware, middleware_cls=binding.middleware, prefixes=binding.prefixes)
```

`ModuleAccessMiddleware` 是一个简单的标记中间件，为匹配的请求设置 `request.state.module_accessed = True`。

### 5.4 路由标签系统

> 源码位置：`app/framework/router/route_meta.py`

通过标签装饰器为路由方法打上元数据标记：

| 标签 | 装饰器 | 说明 |
|------|--------|------|
| `IGNORE_TOKEN` | `@allow_anonymous` | 跳过 Token 校验 |
| `IGNORE_PERMISSION` | — | 跳过权限校验 |
| `IGNORE_LOG` | — | 跳过操作日志 |

## 7. EPS 元数据协议

> 源码位置：`app/framework/eps/scanner.py`、`app/framework/eps/builder.py`、`app/modules/base/service/eps_service.py`

### 7.1 什么是 EPS

EPS (Entity Protocol Specification) 是 Cool-Admin 的核心技术，用于**将后端的实体和路由元数据自动输出给前端**，实现：

- 前端自动生成 API 请求函数
- IDE 智能提示（TypeScript 类型）
- 消除手写 API 封装和类型定义

### 7.2 数据流

```
后端启动 → @CoolController 注册 ExportedRouteMeta
         → build_eps_data() 构建树结构
         → EpsService 聚合 OpenAPI + 自定义路由
         → GET /admin/base/open/eps 输出 JSON

前端启动 → 请求 EPS 接口
         → 解析 JSON 树
         → 自动生成 .d.ts 类型文件
         → 生成 service.base.sys.user.page() 等调用链
```

### 7.3 模型列扫描

`scanner.py` 扫描 SQLModel/Pydantic 模型，提取字段元数据：

```python
scan_model_columns(UserListItem)
# 输出:
[
  {
    "propertyName": "username",
    "source": "username",
    "prop": "username",
    "type": "string",
    "comment": "用户名",
    "label": "用户名",
    "nullable": false,
    "required": true
  },
  {
    "propertyName": "status",
    "type": "select",
    "dict": [{"label": "ACTIVE", "value": 1}, {"label": "DISABLED", "value": 0}]
  }
]
```

类型映射规则：

| Python 类型 | EPS 类型 |
|------------|---------|
| `int`, `float` | `number` |
| `bool` | `boolean` |
| `str` | `string` |
| `datetime`, `date` | `datetime` |
| `Enum` 子类 | `select` (附带 dict 选项) |

验证规则提取：`max_length`, `min_length`, `pattern`, `gt`, `lt`, `ge`, `le`

### 7.4 EPS 树结构

`builder.py` 将注册的路由元数据构建为嵌套树：

```json
{
  "base": {
    "sys": {
      "user": {
        "page": {
          "method": "POST",
          "path": "/admin/base/sys/user/page",
          "summary": "获取分页",
          "permission": "base:sys:user:page",
          "query": {
            "keywordLikeFields": ["username", "name"],
            "fieldEq": [{"column": "status", "requestParam": "status"}]
          }
        }
      }
    }
  }
}
```

- 聚合 FastAPI 原生 OpenAPI 路由
- 处理资源命名冲突
- 类型转换（`integer` → `number` 等）
- 兼容路由前缀映射

## 8. 查询构造器与数据权限

> 源码位置：`app/framework/router/query_builder.py`、`app/modules/base/service/data_scope_service.py`、`app/modules/base/service/admin_service.py`

### 8.1 QueryBuilder

`QueryBuilder` 是通用查询构造器，负责将 `CrudQuery` 对象转化为 SQLAlchemy 查询语句：

```python
builder = QueryBuilder(model=User, query=crud_query)
statement = select(User)
statement = builder.apply_all(
    statement,
    data_scope=data_scope,
    current_user_id=user_id,
    relations=relations,
)
```

### 8.2 查询处理链

`apply_all()` 按以下顺序链式应用过滤规则：

| 步骤 | 方法 | 说明 |
|------|------|------|
| 1 | `apply_soft_delete` | 过滤 `delete_time IS NULL` |
| 2 | `apply_data_scope` | 数据权限隔离 |
| 3 | `apply_filters` | 精确匹配 (eq) / 模糊匹配 (like)，支持 IN |
| 4 | `apply_keyword` | 关键词多字段模糊搜索 (OR) |
| 5 | `apply_ranges` | 时间范围过滤 (`*StartTime`/`*EndTime` 后缀) |
| 6 | `apply_where` | 自定义过滤函数 |
| 7 | `apply_select` | 字段选择（当前仅用于 EPS 导出，不实际限制 SQL） |
| 8 | `apply_relations` | 自动关联查询 (LEFT OUTER JOIN + add_columns) |
| 9 | `apply_sort` | 排序 (显式 order/sort > add_order_by > 兜底 created_at desc) |

### 8.3 数据权限隔离

`apply_data_scope` 基于模型字段自动注入数据权限过滤：

```
- 模型有 user_id 字段 → 允许访问本人创建的数据
- 模型有 department_id 字段 → 允许访问有权限部门的数据
- 两者通过 OR 连接，既可看自己的，也可看有权限部门的
- 超管 (allow_all=True) 跳过所有过滤
```

### 8.4 DataScopeContext

> 源码位置：`app/modules/base/service/data_scope_service.py`

```python
@dataclass
class DataScopeContext:
    allow_all: bool = False
    allowed_department_ids: set[int] = field(default_factory=set)
    scope_level: str = "self"   # all / department / custom / self
```

`resolve_data_scope(user)` 根据用户角色解析数据权限级别：
- **超管 / is_super_admin** → `allow_all=True`
- **角色绑定部门** → 收集允许的部门 ID 及子部门
- **无部门权限** → 仅可访问自己的数据

### 8.5 关联查询

通过 `RelationConfig` 声明式配置关联：

```python
relations=(
    RelationConfig(
        model=Department,           # 目标模型
        column="department_id",     # 当前模型的关联字段
        target_column="name",       # 目标模型中要选取的字段
        alias="departmentName",     # 输出别名
    ),
)
```

自动生成 `LEFT OUTER JOIN` 并 `add_columns` 选取目标字段。
