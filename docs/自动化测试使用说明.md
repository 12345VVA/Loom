# 自动化测试使用说明

Loom 的自动测试分为后端 pytest、前端单元测试、前端类型/构建检查和可选 E2E 冒烟测试。

除 `npm run lint`、`npm run format` 外，本文列出的验证命令不会改写仓库源文件；它们可能生成或更新 `.pytest_cache/`、coverage 临时数据、前端构建产物、Playwright 浏览器缓存等运行产物。

## 后端

后端本地测试应使用项目规定的 `backend/venv` 虚拟环境。首次准备环境时，在仓库根目录执行：

```powershell
cd backend
python -m venv venv
venv\Scripts\activate
pip install -r requirements-dev.txt
```

后续运行测试时，先进入 `backend/` 并激活虚拟环境：

```powershell
cd backend
venv\Scripts\activate
python -m pytest
python -m pytest --cov=app --cov-report=term-missing
```

`pytest.ini` 已限制只收集 `tests/`，不会误收集 `scratch/`。测试依赖应安装到 `backend/venv` 中，不建议直接安装到系统 Python 环境。

当前后端测试重点覆盖：

- 鉴权、登录、验证码、Token 刷新和菜单权限。
- 自动路由、CRUD GET/POST 双协议、EPS 字段映射。
- QueryBuilder、DataScope、字段别名和安全补丁回归。
- 生产化补强能力：事务提交/回滚、启动配置校验、缓存命名空间、导入导出字段校验、metrics 事件和任务调度缓存。

数据库迁移相关验证：

```powershell
cd backend
alembic current
alembic history
```

生产库首次接入迁移时，先确认当前 schema 已与模型一致，再执行：

```powershell
alembic stamp 20260502_0001
```

## 前端

在 `frontend/` 目录执行：

```bash
npm run type-check
npm run build
npm run test:unit
```

单元测试使用 Vitest，测试文件放在 `frontend/tests/unit/**/*.test.ts`。优先测试菜单路由转换、权限判断、store 纯逻辑和 service 包装。

注意：

- `npm run type-check` 只做 TypeScript/Vue 类型检查。
- `npm run build` 会生成前端构建产物。
- `npm run test:unit` 使用 Vitest，通常只写测试缓存。
- `npm run lint` 和 `npm run format` 会改写文件，除非任务需要或已确认，否则不要作为常规只读验证命令运行。

## E2E

E2E 使用 Playwright，测试文件放在 `frontend/tests/e2e/**/*.spec.ts`。

```bash
npm run test:e2e
```

默认会启动前端 dev server，基础地址为 `http://127.0.0.1:9090`。如果要连接已有前端服务：

```bash
PLAYWRIGHT_SKIP_WEB_SERVER=1 PLAYWRIGHT_BASE_URL=http://127.0.0.1:9090 npm run test:e2e
```

后续需要完整登录链路时，先启动后端和 Redis，再扩展 E2E 用例。

## 一键脚本

Windows PowerShell 可在仓库根目录执行：

```powershell
.\scripts\test.ps1
.\scripts\test.ps1 -Coverage
.\scripts\test.ps1 -E2E
```

如果仓库中存在 `backend\venv\Scripts\python.exe`，`scripts/test.ps1` 会优先使用该解释器；否则回退到当前 PATH 中的 `python`。运行 `-Coverage` 前请确认已安装 `requirements-dev.txt`，否则 `pytest-cov` 参数不可用。

脚本执行顺序为：

1. `cd backend; python -m pytest`，加 `-Coverage` 时改为 coverage 模式。
2. `cd frontend; npm run type-check`
3. `cd frontend; npm run build`
4. `cd frontend; npm run test:unit`
5. 加 `-E2E` 时额外运行 `npm run test:e2e`

## 当前阶段

当前阶段只建设本地自动测试，不接入 CI。后续如果需要远端门禁，应单独规划干净环境的 `.env`、Redis、Playwright 浏览器安装和测试数据初始化。
